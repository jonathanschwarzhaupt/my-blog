# home-blog

A personal blog and passive CV for Jonathan, built to make daily writing from a phone effortless while keeping the bar for "why does this matter" high.

## Language

**Post**:
The atomic unit of content: a title, a markdown body, and a required So What. Every piece of writing is a Post regardless of length or topic — there is no separate "note" vs "article" type.

**So What**:
A required one-line field on every post, filled in at compose time, stating why the post matters. Exists to enforce quality on a daily-posting cadence rather than let volume dilute substance.
_Avoid_: Takeaway, summary, TL;DR

**Project**:
A deliberately-curated, named collection of Posts with its own dedicated page (e.g. "Homelab"). Created explicitly, not automatically — few in number by design. A Post can belong to zero or more Projects.
_Avoid_: Series, channel, topic

**Tag**:
A lightweight, free-text label on a Post used for cheap topical filtering (e.g. `go`, `career`). Unlike a Project, a Tag has no dedicated page and requires no upfront curation — created ad hoc while writing.

**Skill**:
A technology or tool shown on the About page (e.g. `Kubernetes`, `Go`), grouped under a free-text category (e.g. "Infrastructure & Platform") set per-Skill — categories aren't a separate curated entity, they emerge from whatever values Skills are given. Ordered the same way as Project: an explicit, editable position, not automatic.

**Revision**:
A full historical snapshot of the About page's content, kept every time it's edited — never overwritten, so any past state can be viewed and restored.
_Avoid_: Version (reserved for the edit-conflict guard Posts and Projects already have, which holds no history — a Revision is a different concept entirely, not a stricter version of it)
